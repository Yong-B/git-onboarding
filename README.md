# Git Onboarding

## 학습 목표

GitHub 저장소를 clone하고, feature 브랜치에서 작업한 뒤 PR과 Self Review를 거쳐 main 브랜치에 병합하는 협업 흐름을 익힌다.

## 작업 단위

자기소개 문서를 아래와 같은 브랜치 단위로 나누어 작성한다.

| 브랜치명 | 작업 내용 | 파일 |
| --- | --- | --- |
| feature/about | 기본 자기소개 작성 | members/about.md |
| feature/skills | 관심 기술 작성 | members/skills.md |
| feature/goals | 온보딩 목표 작성 | members/goals.md |

## PR 진행 순서

1. `feature/about` 브랜치에서 기본 자기소개를 작성하고 PR을 올린다.
2. `feature/skills` 브랜치에서 관심 기술을 작성하고 PR을 올린다.
3. `feature/goals` 브랜치에서 온보딩 목표를 작성하고 PR을 올린다.
4. 각 PR에서 Self Review를 작성한 뒤 main 브랜치에 merge한다.
